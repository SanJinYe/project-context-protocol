# Lifecycle Example

## Scenario
A solo developer starts a demo project, pauses midway, switches tasks, and later prepares the repo for publishing.

## Step 1: init-project
- create requirements/design/progress contract
- classify project mode
- bootstrap minimal environment

## Step 2: init-task
- read README
- read CURRENT_PROGRESS
- read recent RECORD entries
- identify impacted files
- restate current task against current reality

## Step 3: update-repo
- rewrite README for shipped reality
- update CURRENT_PROGRESS and RECORD
- keep local-only files out of publish surface
- prepare commit/push if needed
